## Think of an Agent Like a Trainee Intern

An **Agent** in OMNI is an autonomous AI worker designed to **sense, decide, and act in real time**. The simplest way to grasp an Agent's function is to liken it to an **intern you are training**.

Like an intern, you provide the Agent with everything it needs to perform a task:

<CardGroup cols={3}>
  <Card title="Instructions (System Prompt)" icon="pencil">
    This is the detailed logic and steps for performing the task, telling the
    agent how to think and what to prioritize. The system prompt contains the
    information and steps for the agent.
  </Card>
  <Card title="References (Knowledge Base)" icon="book-open">
    Collections of information, references, and documents the agent can look up
    while working. References are now knowledge bases.
  </Card>
  <Card title="Tools & Access (Integrations)" icon="wrench">
    These are the steps and actions the agent can perform, including the login
    permissions required to act inside external applications like email or
    calendar.
  </Card>
</CardGroup>

<Tip>
  Files you give the agent to reuse often become a **default file**. Files you
  give it to manipulate would either be an **integration**.
</Tip>

## Real-Life Agent Applications

OMNI Agents are capable of executing complex, recurring tasks across various domains. By combining the core components, you can automate essential workflows.

### 💡 Example: Daily Email Summary Agent

This is an agent that automates the process of summarizing your daily emails from specific sources without you needing to open your inbox.

| Component         | Configuration                                                                                                                                                                         | Purpose                                                                |
| :---------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :--------------------------------------------------------------------- |
| **Integrations**  | Connect Outlook or Gmail                                                                                                                                                              | Grants the agent permission to read and access your emails.            |
| **System Prompt** | Direct the agent to summarize emails from specific senders (e.g., `substack, medium, tldr, and every educative email`) and email the summary to a specified address (`xyz@abcd.com`). | Tells the agent _what_ to summarize and _where_ to deliver the output. |
| **Task/Trigger**  | Set a **Schedule Trigger** (e.g., daily at 5:00 PM)                                                                                                                                   | Automates the agent to run at the exact, recurring time you need.      |

### 🛠️ Example: Workflow from the Marketplace

OMNI offers a **Marketplace** where users can download pre-built agents created by others. You can then customize a downloaded agent for your specific needs.

- **Download:** A person who needs to use an agent's skills can download it to their library from the Marketplace.
- **Customize:** If you want the final output slightly changed, you can change the **integrations**, tweak the **system prompt** to adjust the format, or even change the final destination of the task.

## Step-by-Step: Creating an Agent in OMNI

Creating an Agent to automate a task, like the **Daily Email Summaries** example, involves setting up a few core building blocks.

<img
  src="/images/omni_main_page.png"
  alt="Omni main dashboard where you begin agent creation"
/>

<Steps>
  <Step title="1. Create the New Agent">
    Navigate to the **Agents** section on the sidebar and click **New Agent**.

    <img
      src="/images/new-agent-config.png"
      alt="Agent creation modal showing the General setup page"
    />

    Create a new agent, for example: **EMAIL SUMMARIES**.

  </Step>

  <Step title="2. Connect Integrations (Grant Access)">
    In the Agent configuration, go to the **Integrations** tab.

    <img
      src="/images/appbased-trig.png"
      alt="Triggers and integrations view showing schedule trigger"
    />

    Click **Browse Apps** and connect your account (e.g., Outlook or Gmail).

    This grants the agent permission to interact with the app.

  </Step>

  <Step title="3. Configure Tools">
    After connecting an app (like Gmail or Slack), select the specific **tools**
    you want the agent to use.

    <img
      src="/images/slack_trigger.png"
      alt="Slack trigger and profile configuration modal"
    />

    Common examples include:

    - `Modify email labels`
    - `Create email draft`
    - `Send Slack message`
    - `Monitor Slack channel events`

  </Step>

  <Step title="4. Write the System Prompt (Instructions)">
    Go to the **Instructions** tab.

    Edit the system prompt to define the agent's core identity and task flow.

    **Example Prompt:**

    ```text
    Create a summary of emails from substack, medium, tldr and every
    educative email and email it to me at varnika@latent-labs.ai
    ```

  </Step>

  <Step title="5. Set up the Task (Schedule)">
    Go to **Tasks** or the **Triggers** tab and click **Create Schedule Trigger**.

    In the **Task Setup** step, set a **Task Name** and an optional description.

    In the **Schedule Configuration** step, choose how often the task should run
    (**Recurring**, **One-time**, or **Advanced**) and set the time.

    In the **Execution Configuration** step, choose the **Execution Method**
    (**Send Prompt to Agent** is selected by default) and provide the final
    **Agent Instructions**, such as:

    `Run the agent and generate today’s email summary.`

  </Step>
</Steps>

<Check>
  Once this is configured, your agent will follow the exact logic you defined,
  acting like a background worker that runs on your chosen schedule.
</Check>

## Key Tips for Agent Configuration

- **Avoid vague instructions.** Provide **exact action words** instead of being general.
- **Manage tokens.** Try to send a **single clear message** instead of a chain of back-and-forth messages. Every message you send uses up credits/tokens as input.
- **Troubleshooting hallucination.** If your agent misbehaves (what OMNI calls **hallucination**), end the chat and start fresh. Do **not** ask the agent to perform the same task over and over again.
- **Request new integrations.** If the software you want to use is not available in Integrations, asking the agent to perform the task anyway will waste tokens and credits. Instead, reach out to OMNI to request an integration.
- **Generate the system prompt.** If you're unsure about the right system prompt, tell the agent your logic and ask it to help you **generate the SYSTEM PROMPT**.
