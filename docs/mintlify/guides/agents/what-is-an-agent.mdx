## Think of an Agent Like a Trainee Intern

[cite_start]An **Agent** in OMNI is an autonomous AI worker designed to sense, decide, and act in real-time[cite: 2]. [cite_start]The simplest way to grasp an Agent's function is to liken it to an **intern you are training**[cite: 38].

[cite_start]Like an intern, you provide the Agent with everything it needs to perform a task[cite: 39]:

<CardGroup cols={3}>
  <Card title="Instructions (System Prompt)" icon="pencil">
    [cite_start]This is the detailed logic and steps for performing the task,
    telling the agent how to think and what to prioritize[cite: 40].
  </Card>
  <Card title="References (Knowledge Base)" icon="book-open">
    [cite_start]Collections of information, references, and documents the agent
    can look up while working[cite: 39, 40].
  </Card>
  <Card title="Tools & Access (Integrations)" icon="wrench">
    [cite_start]These are the steps and actions the agent can perform, including
    the login permissions required to act inside external applications like
    email or calendar[cite: 43, 44].
  </Card>
</CardGroup>

<Tip>
  [cite_start]Information and steps are considered the **system prompt**[cite:
  40]. [cite_start]Files given to reuse often are called **default files**[cite:
  41].
</Tip>

## Real-Life Agent Applications

OMNI Agents are capable of executing complex, recurring tasks across various domains. By combining the core components, you can automate essential workflows.

### 💡 Example: Daily Email Summary Agent

[cite_start]This is an agent that automates the process of summarizing your daily emails from specific sources without you needing to open your inbox[cite: 63, 161].

| Component         | Configuration                                                                                                                                                                                                          | Purpose                                                                                            |
| :---------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------- |
| **Integrations**  | [cite_start]Connect Outlook or Gmail [cite: 161, 221]                                                                                                                                                                  | [cite_start]Grants the agent permission to read and access your emails[cite: 163].                 |
| **System Prompt** | [cite_start]Direct the agent to summarize emails from specific senders (e.g., `substack, medium, tldr, and every educative email`) and email the summary to a specified address (`varnika@latent-labs.ai`)[cite: 313]. | [cite_start]Tells the agent _what_ to summarize and _where_ to deliver the output[cite: 164, 165]. |
| **Task/Trigger**  | [cite_start]Set a **Schedule Trigger** (e.g., daily at 5:00 PM) [cite: 329, 368]                                                                                                                                       | [cite_start]Automates the agent to run at the exact, recurring time you need it[cite: 61, 166].    |

### 🛠️ Example: Workflow from the Marketplace

[cite_start]OMNI offers a Marketplace where users can download pre-built agents created by others[cite: 85, 106]. [cite_start]You can then customize a downloaded agent for your specific needs[cite: 87].

- [cite_start]**Download:** A person (B) can download an agent released by another person (A) from the Marketplace to their library[cite: 85].
- [cite_start]**Customize:** If you want the final output slightly changed, you can modify the agent's **integrations** or change the **system prompt** to alter the format or final task destination[cite: 86, 87].

## Step-by-Step: Creating an Agent in OMNI

[cite_start]Creating an Agent to automate a task, like the "Email Summaries" example, involves setting up the core building blocks[cite: 167].

<Steps>
  <Step title="1. Create the New Agent">
    * [cite_start]Navigate to the **Agents** section and click **New Agent** or **+ Create Agent**[cite: 8, 11, 179].
    * [cite_start]Give your new agent a name, for example: **EMAIL SUMMARIES**[cite: 168].
      </Step>

{" "}
<Step title="2. Connect Integrations (Grant Access)">
  * [cite_start]In the Agent configuration, go to the **Integrations** tab[cite:
  55, 214]. * [cite_start]Click **Browse Apps** and connect your account (e.g.,
  Outlook or Gmail) to give the agent permission to interact with the app[cite:
  125, 204, 221]. [cite_start]This is the only place you should provide login
  authorization[cite: 412, 413].
</Step>

{" "}
<Step title="3. Configure Tools">
  * [cite_start]After connecting an app (like Gmail), select the specific
  **tools** you want the agent to use (e.g., `Modify email labels`, `Create
  email draft`)[cite: 265, 270, 275, 287]. * [cite_start]A connected app may
  enable a certain number of tools for your agent (e.g., 13 tools enabled)[cite:
  285, 297].
</Step>

{" "}
<Step title="4. Write the System Prompt (Instructions)">
  * [cite_start]Go to the **Instructions** tab[cite: 51, 292]. *
  [cite_start]Edit the system prompt to define the agent's core identity and
  give it the exact, logical instructions for the task[cite: 308, 310, 311,
  422]. * [cite_start]**Example Prompt:** `Create a summary of emails from
  substack, medium, tldr and every educative email and email it to me
  @varnika@latent-labs.ai`[cite: 307, 313].
</Step>

  <Step title="5. Set up the Task (Schedule)">
    * [cite_start]Go to the **Triggers** tab and click **Create Schedule Trigger**[cite: 58, 76, 339].
    * [cite_start]In the **Create Scheduled Task** flow, set a **Task Name**[cite: 341, 351].
    * [cite_start]In the **Schedule** step, choose how often the task should run (e.g., **Recurring** or **One-time**) and set the time[cite: 367, 368, 369, 374].
    * [cite_start]In the **Execute** step, choose the **Execution Method** (e.g., **Send Prompt to Agent**) and provide the final **Agent Instructions** to run the task[cite: 394, 398, 400].
      </Step>
</Steps>

<Check>
  [cite_start]By following these steps, you define the agent's capabilities and
  its automated schedule, turning it into a background worker[cite: 347, 365].
</Check>

## Key Tips for Agent Configuration

- [cite_start]**Avoid Vague Instructions:** Always provide **exact action words** in your instructions instead of being vague[cite: 422].
- [cite_start]**Manage Tokens:** Send a singular, well-defined message to the agent, as sending a chain of multiple messages uses up tokens/credits for every input, regardless of the screen showing an output[cite: 417, 419].
- [cite_start]**Troubleshooting:** If the agent _misbehaves_ (hallucinates) or you are confused, end the chat and do **not** ask the agent to perform the task repeatedly[cite: 414, 415].
- **Request Integrations:** If the software you need is not available, request a new integration. [cite_start]Do not ask the agent to perform the task without the correct tool, as this wastes tokens and credits[cite: 405, 406, 409].
