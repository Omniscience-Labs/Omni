## System Prompt Engineering

Craft effective prompts that shape your agent's behavior.

### Prompt Structure

<Steps>
  <Step title="Define Role & Expertise">
    Start with a clear role definition:

    ```
    You are a [specific role] specializing in [domain/expertise].
    ```

    Example:
    ```
    You are a senior content strategist specializing in B2B SaaS 
    marketing with expertise in SEO and conversion optimization.
    ```
  </Step>

  <Step title="List Core Competencies">
    Enumerate specific capabilities:

    ```
    Your core competencies include:
    - [specific skill 1]
    - [specific skill 2]
    - [specific skill 3]
    ```

    <Tip>
      Be specific rather than general. "Analyze customer feedback data using sentiment analysis" is better than "analyze data".
    </Tip>
  </Step>

  <Step title="Define Behavior Guidelines">
    Specify how the agent should approach tasks:

    ```
    When working on tasks:
    - [guideline about process]
    - [guideline about quality]
    - [guideline about format]
    ```

    Example:
    ```
    When creating content:
    - Research target audience pain points first
    - Use clear, jargon-free language
    - Include actionable takeaways
    - Optimize for SEO with natural keyword integration
    ```
  </Step>

  <Step title="Set Constraints & Rules">
    Add important limitations or requirements:

    ```
    Always [critical requirement].
    Never [prohibited action].
    If [condition], then [action].
    ```

    Example:
    ```
    Always cite sources when presenting statistics or research.
    Never use discriminatory or biased language.
    If creating technical content, include code examples.
    ```
  </Step>
</Steps>

### Prompt Examples

<Tabs>
  <Tab title="Customer Support">
    ```
    You are a customer support specialist focused on providing 
    exceptional service with empathy and efficiency.

    Your expertise includes:
    - Troubleshooting common product issues
    - Explaining features clearly to non-technical users
    - Escalation procedures for complex problems
    - Creating help documentation

    When assisting customers:
    - Begin by acknowledging their concern with empathy
    - Ask clarifying questions before providing solutions
    - Provide step-by-step instructions with verification
    - Follow up to ensure the issue is resolved

    Always maintain a friendly, professional tone. Never make 
    promises about features or timelines without confirmation.
    ```
  </Tab>

  <Tab title="Code Reviewer">
    ```
    You are a senior software engineer specializing in code 
    review with expertise in clean code principles, security, 
    and performance optimization.

    Your review process includes:
    - Code quality and maintainability assessment
    - Security vulnerability identification
    - Performance bottleneck detection
    - Best practice verification

    When reviewing code:
    - Provide specific, actionable feedback with examples
    - Explain *why* changes improve the code
    - Suggest alternative approaches when applicable
    - Highlight positive patterns worth preserving

    Always prioritize security and data integrity. If you 
    identify critical security issues, flag them immediately.
    ```
  </Tab>

  <Tab title="Marketing Analyst">
    ```
    You are a marketing data analyst specializing in campaign 
    performance analysis and strategic recommendations.

    Your analytical framework covers:
    - Multi-channel attribution modeling
    - ROI and conversion rate optimization
    - Audience segmentation analysis
    - Competitive benchmarking

    When analyzing campaigns:
    - Identify trends and patterns in the data
    - Segment performance by channel and audience
    - Calculate key metrics (CAC, LTV, ROAS)
    - Provide actionable optimization recommendations

    Always present findings with clear visualizations. Support 
    recommendations with data-driven reasoning and include 
    expected impact estimates.
    ```
  </Tab>
</Tabs>

## Integration Configuration

Connect external services to extend agent capabilities.

### Adding Integrations

<Steps>
  <Step title="Select Service">
    Navigate to the **Integrations** tab and browse available services.

    <CardGroup cols={3}>
      <Card title="Gmail" icon="envelope">
        Read, send, manage emails
      </Card>
      <Card title="Slack" icon="slack">
        Send messages, read channels
      </Card>
      <Card title="GitHub" icon="github">
        Manage repos, issues, PRs
      </Card>
    </CardGroup>
  </Step>

  <Step title="Authenticate">
    Click **Connect** and complete OAuth flow:
    1. Log in to the service
    2. Review permissions
    3. Authorize access

    <Info>
      You'll be redirected back to Omni after successful authentication.
    </Info>
  </Step>

  <Step title="Configure Permissions">
    Select which actions the agent can perform:
    - Read-only access
    - Create/modify content
    - Delete permissions (optional)

    {/* 
      ðŸ“¸ MEDIA PLACEHOLDER: Permission Configuration
      - Checklist showing available permissions
      - Each permission with explanation
      - Recommended vs advanced options highlighted
      - Test connection button
      - Size: 1400x800px
    */}
  </Step>

  <Step title="Test Connection">
    Verify the integration works correctly:

    ```
    Test Task: "Check my unread Gmail messages"
    Expected: Agent retrieves and summarizes emails
    ```

    <Check>
      Integration is active and ready to use!
    </Check>
  </Step>
</Steps>

### Managing Integrations

<AccordionGroup>
  <Accordion title="Health Monitoring" icon="heart-pulse">
    Track integration status:
    - **Active**: Connected and functioning
    - **Warning**: Potential issues detected
    - **Error**: Connection failed or expired
    - **Disabled**: Manually turned off

    View detailed logs to troubleshoot problems.
  </Accordion>

  <Accordion title="Permission Updates" icon="key">
    Modify access as needs change:
    - Expand permissions for new use cases
    - Restrict access for security
    - Rotate credentials periodically
    - Revoke access when no longer needed
  </Accordion>

  <Accordion title="Usage Analytics" icon="chart-simple">
    Monitor integration activity:
    - API calls per day/week/month
    - Most used endpoints
    - Error rates
    - Response times

  </Accordion>

## Knowledge Base Configuration

Control what information your agent can access.

### Access Levels

<Tabs>
  <Tab title="Global Knowledge">
    **Company-Wide Information**:
    - Accessible to all agents
    - Centrally managed
    - Automatically updated
    - Version controlled

    Enable for agents that need general company context.
  </Tab>

  <Tab title="Agent-Specific">
    **Dedicated Resources**:
    - Exclusive to this agent
    - Customized content
    - Specialized documentation
    - Domain-specific data

    Upload files directly in agent settings.
  </Tab>

  <Tab title="Project-Based">
    **Contextual Access**:
    - Scoped to specific projects
    - Temporary or permanent
    - Shareable across team agents
    - Auto-archived when project completes
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Agent Not Responding as Expected" icon="triangle-exclamation">
    **Potential Causes**:
    - Unclear system prompt
    - Wrong tools enabled
    - Knowledge base not accessible
    - Model selection mismatch

    **Solutions**:
    1. Refine system prompt with specific examples
    2. Verify required tools are enabled
    3. Check knowledge base permissions
    4. Try a more capable model
  </Accordion>

  <Accordion title="Integration Errors" icon="plug-circle-xmark">
    **Common Issues**:
    - **401 Unauthorized**: Credentials expired
    - **403 Forbidden**: Insufficient permissions
    - **429 Rate Limit**: Too many requests
    - **500 Server Error**: Service unavailable

    **Resolution Steps**:
    1. Reauth enticate the connection
    2. Update permission settings
    3. Wait and retry later
    4. Check service status page
  </Accordion>
