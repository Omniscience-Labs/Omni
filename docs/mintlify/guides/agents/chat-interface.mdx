## Input Methods

<Tabs>
  <Tab title="Text Input" icon="keyboard">
    **Type Your Requests**:
    - Click the input field at bottom of screen
    - Type your request or task
    - Press `Enter` to send or click the send button
    - Use `Shift + Enter` for new lines
    
    Most precise for complex, detailed requests.
  </Tab>
  
  <Tab title="Voice Recording" icon="microphone">
    **Speak Your Requests**:
    1. Click the **microphone button** next to the send button
    2. Speak clearly while recording indicator shows
    3. Click **Stop Recording** when finished
    4. Audio automatically transcribed and sent
    
    <Tip>
      Voice recording works best in quiet environments. The system transcribes your speech into text before processing.
    </Tip>
    
    <Warning>
      **Note**: Voice recording may not work properly during video calls as the microphone is in use. Use text input if you're in a meeting.
    </Warning>
  </Tab>
  
  <Tab title="File Attachments" icon="paperclip">
    **Upload Files with Requests**:
    - Click attachment icon (üìé) in input area
    - Select files from your computer (up to 500MB each)
    - Drag and drop also supported
    - Multiple files can be attached at once
    
    **Larger Files**: For files over 500MB, connect OneDrive or cloud storage integration to access them directly.
  </Tab>
</Tabs>

## Chat Management

<Tabs>
  <Tab title="Creating Chats" icon="plus">
    **Start New Conversations**:
    - Type in the input field to automatically create a new chat
    - Or start typing with a specific agent selected
    - Each chat is independent with its own workspace and files
    
    <Info>
      You can have up to **500 chats** per user. Delete old chats when you reach the limit.
    </Info>
  </Tab>

  <Tab title="Organizing Chats" icon="list">
    **Left Sidebar Chat List**:
    - All your chats appear chronologically
    - Today's chats listed first
    - Past month's chats below
    - Hover over any chat to see action menu
    
    **Auto-Naming**: Chats are named based on your first message. Rename them for clarity:
    ```
    Good: "Electrical Quotation - Smith Project"
    Poor: "Google search image related"
    ```
  </Tab>

  <Tab title="Deleting Chats" icon="trash">
    **Clean Up Your Workspace**:
    1. Hover over a chat in the left sidebar
    2. Click the checkbox that appears
    3. Select multiple chats if needed
    4. Click **Delete** to remove them
    
    <Warning>
      **Deleted chats cannot be recovered**. Download any important files first.
    </Warning>
    
    **Bulk Delete**: Select multiple chats at once to delete in batch when you approach the 500 chat limit.
  </Tab>
  
  <Tab title="Renaming Chats" icon="pen">
    **Keep Chats Organized**:
    - Hover over a chat in the sidebar
    - Click to select it
    - Look for rename option in action menu
    - Give it a descriptive name
    
    <Tip>
      Rename chats immediately while context is fresh. Use naming patterns like "[Project] - [Task Type]" for easy searching later.
    </Tip>
  </Tab>
</Tabs>

## Accessing Chats on Mobile

Omni's full chat interface works on mobile web browsers:

<Steps>
  <Step title="Open on Mobile Browser">
    Navigate to your organization's Omni URL on your phone's browser (Safari, Chrome, etc.)
  </Step>
  
  <Step title="Sign In with SSO">
    Use the same **Continue with Microsoft** or Google login process
  </Step>
  
  <Step title="Switch Agents on Mobile">
    **Agent Switcher Location** (different from desktop):
    - Look at the text input area at the bottom
    - Find the dropdown button **left of the microphone icon**
    - Tap it to see list of available agents
    - Select the agent you want to use
    
    <Info>
      On mobile, the agent switcher is integrated into the input bar for easy thumb access.
    </Info>
  </Step>
  
  <Step title="Use Voice Input">
    Tap the **microphone button** to speak your requests - particularly useful on mobile devices
  </Step>
</Steps>

## Chat Lifecycle & Limitations

<Warning>
  **Inactive Chat Behavior**: Chats older than 30 days may experience issues:
  - Conversation history remains accessible
  - You can continue chatting in old threads
  - **Files may fail to load** in very old chats
  
  **Workaround**: If files won't load in an old chat:
  1. Copy the text of your original request
  2. Create a new chat with the same agent
  3. Paste the request to get fresh results
  4. Files will be accessible in the new chat
</Warning>

<Info>
  **Chat Workspace**: Each chat has its own isolated workspace (like its own computer). Files created in one chat are separate from other chats. This keeps your work organized but means you can't directly reference files across chats.
</Info>

## Crafting Effective Requests

The quality of your request directly impacts results.

### Request Structure

<Steps>
  <Step title="Be Specific">
    **Less Effective**:
    ```
    "Write a blog post"
    ```

    **More Effective**:
    ```
    "Write a 1200-word blog post about sustainable manufacturing 
    for B2B decision-makers. Include:
    - Current industry challenges
    - 3 innovative solutions with case studies
    - Implementation roadmap
    - ROI considerations
    Target tone: Professional yet accessible"
    ```

    {/* 
      üì∏ MEDIA PLACEHOLDER: Request Comparison
      - Side-by-side showing vague vs specific requests
      - Highlight the differences in detail
      - Show resulting outputs side by side
      - Size: 1600x800px
    */}
  </Step>

  <Step title="Provide Context">
    Help the agent understand background:
    ```
    "Context: We're launching a new API for developers.
    
    Task: Create comprehensive API documentation including:
    - Authentication guide
    - 5 common use case examples with code
    - Error handling best practices
    
    Audience: JavaScript developers, intermediate level
    Format: Markdown with code blocks"
    ```

    <Info>
      More context = better results. Include relevant background information upfront.
    </Info>
  </Step>

  <Step title="Specify Desired Format">
    Tell the agent how to structure output:
    - "Create a table comparing..."
    - "Generate a bulleted list of..."
    - "Write in Q&A format..."
    - "Produce a step-by-step guide..."
    - "Format as JSON with..."

    Formats the agent understands:
    - Tables, lists, outlines
    - Code blocks with language
    - Markdown, HTML, JSON
    - Charts and visualizations
  </Step>

  <Step title="Include Examples">
    Show what "good" looks like:
    ```
    "Write product descriptions similar to this example:
    
    [paste example]
    
    Maintain the same tone, structure, and level of detail
    for these products: [list]"
    ```
  </Step>
</Steps>

### Request Patterns

<Tabs>
  <Tab title="Research Tasks">
    ```
    "Research [topic] focusing on:
    - [aspect 1]
    - [aspect 2]
    - [aspect 3]
    
    Sources: [specify if needed]
    Depth: [summary vs comprehensive]
    Format: [report, presentation, brief]
    Deadline context: [if time-sensitive]"
    ```
  </Tab>

  <Tab title="Content Creation">
    ```
    "Create [content type] about [topic]
    
    Specifications:
    - Length: [word count / pages]
    - Tone: [professional, casual, technical]
    - Audience: [who will read this]
    - Key messages: [main points to convey]
    - SEO keywords: [if applicable]
    - Call to action: [desired reader action]"
    ```
  </Tab>

  <Tab title="Data Analysis">
    ```
    "Analyze [dataset/file] to:
    - Identify [specific patterns]
    - Calculate [specific metrics]
    - Compare [dimensions]
    - Visualize [specific charts]
    
    Output: [report format, dashboard, summary]
    Focus areas: [what matters most]"
    ```
  </Tab>

  <Tab title="Problem Solving">
    ```
    "Problem: [describe issue]
    
    Context: [relevant background]
    Constraints: [limitations to consider]
    Success criteria: [what "solved" means]
    
    Please provide:
    - Root cause analysis
    - 3 potential solutions with pros/cons
    - Recommended approach with rationale"
    ```
  </Tab>
</Tabs>

### Working with Generated Files

All files created by the agent appear in the **Files panel** on the right side of the chat interface.

    <Info>

## Interactive Features

### Interrupting or Redirecting

<AccordionGroup>
  <Accordion title="Stop Current Task" icon="stop">
    Click **Stop** button if:
    - Task is taking wrong direction
    - You need to add information
    - Requirements changed
    - Results not as expected

    Agent stops immediately and awaits new instructions.
  </Accordion>

  <Accordion title="Provide Mid-Task Feedback" icon="comments">
    Send messages while agent works:
    ```
    "Actually, focus more on the competitive analysis 
    section and skip the historical background."
    ```

    Agent adjusts approach in real-time.
  </Accordion>

  <Accordion title="Ask Clarifying Questions" icon="circle-question">
    Agent may ask for guidance:
    ```
    Agent: "I found multiple interpretations of [term]. 
    Which meaning should I use:
    1. [Option A]
    2. [Option B]"
    ```

    Your choice guides the rest of the task.
  </Accordion>
</AccordionGroup>

### Follow-Up Requests

Continue the conversation naturally:

```
You: "Great! Now create a presentation based on this report"

You: "Can you make the charts more colorful?"

You: "Export this as a PDF and send me a shareable link"
```

<Tip>
  **Context is Maintained**: Agent remembers all previous work in the thread. No need to repeat information.
</Tip>

## Common Scenarios

<AccordionGroup>
  <Accordion title="Agent Isn't Understanding" icon="triangle-exclamation">
    **If results miss the mark**:
    1. Provide more specific details
    2. Include an example of what you want
    3. Break complex tasks into smaller steps
    4. Ask the agent to explain its plan first

    ```
    "Let me clarify what I need. I want [specific outcome] 
    formatted like [example]. The key difference from what 
    you provided is [specific change needed]."
    ```
  </Accordion>

  <Accordion title="Task Taking Too Long" icon="clock">
    **For lengthy operations**:
    - Agent shows progress updates
    - You can continue other work
    - Return to check status anytime
    - Receive notification when complete

    Normal for: Large datasets, extensive research, complex analysis
  </Accordion>

  <Accordion title="Unexpected Results" icon="exclamation">
    **When output isn't what you expected**:
    ```
    "This isn't quite right. I was expecting [describe]
    but got [what you received]. Can you revise to 
    focus on [specific aspects]?"
    ```

    Agent will adjust based on your feedback.
  </Accordion>

  <Accordion title="Want to Start Over" icon="rotate-left">
    **To reset a task**:
    1. Click **New Thread** for fresh start
    2. Or say: "Let's start over with a different approach"
    3. Previous work remains in original thread

    Original files and context preserved.
  </Accordion>
</AccordionGroup>
