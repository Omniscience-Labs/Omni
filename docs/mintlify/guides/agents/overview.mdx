## What Are Agents?

Agents are autonomous AI workers that can perform complex tasks on your behalf. Each agent is configured with specific capabilities, knowledge, and integrations tailored to its purpose.

## Agent Components

Every agent consists of four key elements:

### 1. Configuration

The agent's identity and behavior settings:

- **Name & Description**: What the agent is called and its purpose
- **System Prompt**: Core instructions that guide the agent's behavior
- **Icon & Color**: Visual identity for easy recognition
- **Model Selection**: Which AI model powers the agent

  **Pro Tip**: Write clear, specific system prompts. Example: "You are a research assistant specialized in competitive analysis. Always cite sources and provide data-backed insights."
</Tip>

### 2. Tools & Capabilities

The actions your agent can perform:

  <Tab title="Core Tools">

    - Context management

    - Web search and research
    - File creation and editing
    - Browser automation
    - Code execution
    - Data analysis

  <Tab title="Integration Tools">

    - GitHub (repos, issues, PRs)

    - Jira (ticket tracking)

### 3. Knowledge Base Access

What information the agent can reference:

- **Global Knowledge**: Shared company documents available to all agents
- **Agent-Specific**: Files and folders designated for this agent only
- **Semantic Search**: AI-powered search finds relevant info automatically

## Types of Agents

Omni supports different agent configurations for various use cases:

### General Purpose Agents

Versatile agents configured with broad capabilities.

**Best for:**
- Ad-hoc tasks and requests
- Exploratory work
- Teams that need flexibility
- When you're not sure what you'll need

### Specialized Agents

Focused agents optimized for specific domains or tasks.

**Examples:**
- **Research Assistant**: Web search, data analysis, report generation
- **Content Creator**: Writing, editing, SEO optimization
- **Data Analyst**: CSV processing, visualization, statistical analysis
- **Customer Support**: Email management, ticket responses, FAQ lookup

## Agent States

Agents can be in different states:

<Steps>
  <Step title="Active">
    Ready to receive tasks and execute work. This is the normal operating state.

      ðŸ“¸ MEDIA PLACEHOLDER: Active Agent Indicator

  </Step>
  
  <Step title="Working">
    Currently executing a task. You'll see real-time progress updates.
    
    {/*

  </Step>
  
  <Step title="Paused">
    Temporarily disabled. No new tasks will be accepted until unpaused.
  </Step>
  
  <Step title="Error">
    Something went wrong. Check the logs or contact support.
    
    <Warning>
      If an agent enters error state, check for missing integrations, expired credentials, or configuration issues.
    </Warning>
  </Step>
</Steps>
