## Best Practices

<CardGroup cols={2}>
  <Card title="Organize Before Upload" icon="folder-tree">
    Create folder structure first, then upload files into appropriate locations
  </Card>
  <Card title="Descriptive Filenames" icon="signature">

  </Card>
  <Card title="Check File Quality" icon="magnifying-glass-chart">

## Handling Special Cases

<AccordionGroup>
  <Accordion title="Scanned Documents" icon="scanner">
    **PDFs from Scanners**:
    - OCR automatically applied
    - Text extracted with high accuracy
    - Processing takes longer (30-60 seconds)
    - Quality depends on scan resolution

    **Tip**: Scan at 300 DPI or higher for best results.
  </Accordion>

  <Accordion title="Password-Protected Files" icon="lock">
    **Encrypted Documents**:
    - Upload normally
    - Enter password when prompted
    - Password stored securely
    - Content decrypted for indexing

    <Warning>
      Remove password protection if files don't contain sensitive data for easier processing.
    </Warning>
  </Accordion>

  <Accordion title="Large Files" icon="file-lines">
    **Files Over 50MB**:
    - Upload may take several minutes
    - Progress bar shows status
    - Keep browser tab open
    - Automatic resume if interrupted

    **Maximum Size**: 100MB per file (Enterprise plan)

    Consider splitting very large files for better performance.
  </Accordion>

  <Accordion title="Unsupported Formats" icon="file-circle-xmark">
    **If Format Not Supported**:
    - Convert to PDF or TXT
    - Use online converters
    - Export from original application
    - Contact support for format requests

    Most common formats supported.
  </Accordion>
</AccordionGroup>

## Verification & Quality Check

<Steps>
  <Step title="Preview the File">
    Click file name to open preview:
    - Check text extraction quality
    - Verify all pages present
    - Ensure images included
    - Review formatting

    {/* 
      ðŸ“¸ MEDIA PLACEHOLDER: File Preview
      - Split view showing original and extracted text
      - Navigation controls
      - Text highlighting
      - Metadata panel
      - Size: 1800x1000px
    */}
  </Step>

  <Step title="Test Search">
    Search for known content:
    ```
    "Find mentions of [specific term]"
    ```

    Verify results include your uploaded file.
  </Step>

  <Step title="Agent Test">
    Ask agent about the content:
    ```
    "What does [document name] say about [topic]?"
    ```

    Agent should cite the document in response.

    <Check>
      If agent references your document correctly, upload was successful!
    </Check>
  </Step>

  <Step title="Fix Issues if Needed">
    **Common Problems**:
    - **Poor OCR**: Re-scan at higher resolution
    - **Missing Pages**: Re-upload complete file
    - **Formatting Issues**: Convert to different format
    - **Not Searchable**: Check file wasn't corrupted

    Delete and re-upload if problems persist.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Upload Failed" icon="triangle-exclamation">
    **Possible Causes**:
    - File size exceeds limit
    - Network connection interrupted
    - Unsupported file format
    - Storage quota exceeded

    **Solutions**:
    1. Check file size and format
    2. Verify internet connection
    3. Try uploading again
    4. Contact support if persists
  </Accordion>

  <Accordion title="Processing Stuck" icon="spinner">
    **If Processing Takes Too Long**:
    - Refresh the page
    - Check file isn't corrupted
    - Try smaller files first
    - Contact support after 5 minutes

    Most files process within 30 seconds.
  </Accordion>

  <Accordion title="Can't Find Uploaded File" icon="question">
    **Check**:
    - Correct folder location
    - Filter settings (showing all files)
    - Search by filename
    - Recent uploads section

    Use global search if still missing.
  </Accordion>
</AccordionGroup>
