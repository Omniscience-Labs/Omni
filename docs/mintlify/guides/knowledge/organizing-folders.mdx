## Folder Organization Principles

A well-organized Knowledge Base helps both humans and AI find information quickly.

## Creating Folders

<Steps>
  <Step title="Navigate to Parent Location">
    Go to where you want to create the folder:
    - Root of Knowledge Base
    - Inside existing folder
    - Within project

    {/* 
      📸 MEDIA PLACEHOLDER: Folder Creation
      - Knowledge Base interface with "New Folder" button
      - Context menu showing create option
      - Folder name input dialog
      - Size: 1400x800px
    */}
  </Step>

  <Step title="Click New Folder">
    Use **New Folder** button or right-click → **Create Folder**
  </Step>

  <Step title="Name Your Folder">
    **Good Names**:
    ```
    ✅ "Product Documentation"
    ✅ "Q4 2024 Campaign"
    ✅ "Employee Onboarding"
    ✅ "API Reference v2"
    ```

    **Poor Names**:
    ```
    ❌ "Stuff"
    ❌ "Misc"
    ❌ "New Folder"
    ❌ "Documents"
    ```

    <Tip>
      Use descriptive names that explain the folder's purpose at a glance.
    </Tip>
  </Step>

  <Step title="Add Description (Optional)">
    Provide context:
    - What belongs in this folder
    - Who should use it
    - When it's relevant
    - Related folders

    Helps teammates understand organization.
  </Step>

  <Step title="Set Permissions">
    Control access:
    - **Inherit**: Use parent folder permissions
    - **Custom**: Set specific access rules
    - **Public**: All users and agents
    - **Private**: Restricted access

## Folder Best Practices

<AccordionGroup>
  <Accordion title="Limit Nesting Depth" icon="layer-group">
    **Recommended**: 3-4 levels maximum

    **Good** (3 levels):
    ```
    /Marketing
      /Campaigns
        /Q4-Product-Launch
    ```

    **Too Deep** (5+ levels):
    ```
    /Company
      /Departments
        /Marketing
          /Digital
            /Social Media
              /Twitter
                /2024
                  /Q4
    ```

    Shallow structures are easier to navigate.
  </Accordion>

  <Accordion title="Use Consistent Naming" icon="text">
    **Choose a Convention**:

    **Option 1 - Title Case**:
    ```
    /Product Documentation
    /User Guides
    /Release Notes
    ```

    **Option 2 - kebab-case**:
    ```
    /product-documentation
    /user-guides
    /release-notes
    ```

    Stick with one style throughout.
  </Accordion>

  <Accordion title="Add README Files" icon="file-lines">
    **In Each Major Folder**:
    ```markdown
    # Product Documentation

    ## Purpose
    User-facing documentation for all our products.

    ## Contents
    - User Guides: Step-by-step instructions
    - API Docs: Technical reference
    - FAQs: Common questions

    ## Maintenance
    Updated weekly by Product team.
    Contact: docs@company.com
    ```

    Helps new team members understand structure.
  </Accordion>

  <Accordion title="Archive Old Content" icon="box-archive">
    **Create Archive Structure**:
    ```
    /Archived
      /2023
        /Q1
        /Q2
        /Q3
        /Q4
      /2024
    ```

    Move outdated content out of active folders quarterly.
  </Accordion>
</AccordionGroup>

## Moving and Reorganizing

<Tabs>
  <Tab title="Move Files">
    **Drag and Drop**:
    1. Select file(s)
    2. Drag to target folder
    3. Drop to move

    **Cut and Paste**:
    1. Right-click file → Cut
    2. Navigate to destination
    3. Right-click → Paste

    {/* 
      🎥 VIDEO PLACEHOLDER: Moving Files (15 seconds)
      - Show drag and drop operation
      - Multiple file selection
      - Drop target highlighting
      - Success confirmation
    */}
  </Tab>

  <Tab title="Merge Folders">
    **Combining Folders**:
    1. Select all files from source folder
    2. Move to target folder
    3. Delete empty source folder
    4. Update references if needed

    <Warning>
      Check for duplicate filenames before merging.
    </Warning>
  </Tab>

  <Tab title="Bulk Operations">
    **Multi-Select Actions**:
    - `Cmd/Ctrl + Click`: Select individual items
    - `Shift + Click`: Select range
    - `Cmd/Ctrl + A`: Select all in folder

    Then drag, cut, or use bulk actions menu.
  </Tab>
</Tabs>

## Folder Metadata

<AccordionGroup>
  <Accordion title="Tags" icon="tags">
    **Add Folder Tags**:
    - Department: `#marketing`, `#sales`
    - Type: `#documentation`, `#templates`
    - Status: `#active`, `#archived`
    - Priority: `#essential`, `#reference`

    Tags inherited by contained files.
  </Accordion>

  <Accordion title="Access Rules" icon="shield">
    **Folder-Level Permissions**:
    - Set once, apply to all contents
    - New files inherit automatically
    - Override at file level if needed
    - Review access quarterly

    Simplifies permission management.
  </Accordion>

  <Accordion title="Sync Settings" icon="rotate">
    **For Cloud-Synced Folders**:
    - Sync frequency (hourly, daily)
    - Conflict resolution rules
    - File type filters
    - Size limitations

    Configure per folder as needed.
  </Accordion>
</AccordionGroup>

## Common Organizational Patterns

<CardGroup cols={2}>
  <Card title="Status-Based" icon="signal">
    Folders by document status

## Troubleshooting

<AccordionGroup>
  <Accordion title="Folder Structure Too Complex" icon="diagram-nested">
    **Signs**:
    - 5+ levels deep
    - Hard to find files
    - Duplicate folders
    - Unclear naming

    **Solution**: Flatten structure and use tags instead of nested folders.
  </Accordion>

  <Accordion title="Files in Wrong Locations" icon="location-crosshairs">
    **Prevention**:
    - Clear folder descriptions
    - README files explaining structure
    - Team training on organization
    - Regular audits

    **Fixing**: Schedule reorganization sprint, update all references.
  </Accordion>

  <Accordion title="Permission Issues" icon="triangle-exclamation">
    **Common Problems**:
    - Users can't find files they should access
    - Agents can't reference needed documents
    - Overly restrictive permissions

    **Solution**: Review and simplify permission structure.
  </Accordion>
</AccordionGroup>
