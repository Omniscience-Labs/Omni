## Inviting Team Members

<Steps>
  <Step title="Open Project Settings">
    Click gear icon (‚öôÔ∏è) in project header or use `Cmd/Ctrl + ,`

    {/* 
      üì∏ MEDIA PLACEHOLDER: Project Settings Access
      - Settings icon highlighted in UI
      - Settings menu open
      - Sharing tab selected
      - Size: 1400x900px
    */}
  </Step>

  <Step title="Click Share Project">
    In the **Sharing** tab, click **Invite Team Member**
  </Step>

  <Step title="Enter Email Address">
    Add teammate's email:
    - One at a time or multiple comma-separated
    - Selects from your org directory
    - Sends invitation automatically

    <Tip>
      You can also share a direct link instead of sending email invitations.
    </Tip>
  </Step>

  <Step title="Set Permission Level">
    Choose access level:
    - **Viewer**: Read-only access
    - **Commenter**: Can add comments and suggestions
    - **Editor**: Can create and edit threads
    - **Admin**: Full control including sharing

    <Warning>
      Admin access allows removing other members and changing project settings.
    </Warning>
  </Step>

  <Step title="Send Invitation">
    Click **Send Invite** and team member receives:
    - Email notification
    - Direct link to project
    - Overview of their access level

    <Check>
      Team member added! They can access the project immediately upon accepting.
    </Check>
  </Step>
</Steps>

## Permission Levels

<Tabs>
  <Tab title="Viewer" icon="eye">
    **Can**:
    - View all threads and messages
    - Read files and outputs
    - Export conversations
    - See project activity

    **Cannot**:
    - Create or edit threads
    - Upload files
    - Delete content
    - Change settings

    **Best For**: Stakeholders, clients, observers
  </Tab>

  <Tab title="Commenter" icon="comment">
    **Can**:
    - Everything Viewers can do
    - Add comments to threads
    - React to messages
    - Suggest changes
    - Mark items for review

    **Cannot**:
    - Create new threads
    - Edit existing threads
    - Modify files
    - Invite others

    **Best For**: Reviewers, QA team, managers
  </Tab>

  <Tab title="Editor" icon="pen">
    **Can**:
    - Everything Commenters can do
    - Create new threads
    - Send messages to agents
    - Upload and edit files
    - Archive threads
    - Use all agent tools

    **Cannot**:
    - Change project settings
    - Manage team access
    - Delete project

    **Best For**: Team members, contributors, collaborators
  </Tab>

  <Tab title="Admin" icon="crown">
    **Can**:
    - Everything Editors can do
    - Change project settings
    - Manage team access
    - Remove team members
    - Delete project
    - Transfer ownership

    **Best For**: Project leads, team managers
  </Tab>
</Tabs>

## Managing Team Access

### Viewing Current Members

<Tabs>
  <Tab title="Team List">
    See all project collaborators:
    - Name and avatar
    - Email address
    - Permission level
    - Last active time
    - Invitation status (pending/active)

    Sort by name, role, or activity.
  </Tab>

  <Tab title="Activity Log">
    Track team actions:
    - Thread creation
    - File uploads
    - Message sends
    - Setting changes
    - Member additions/removals

    Filterable by member and date.
  </Tab>
</Tabs>

### Changing Permissions

<Steps>
  <Step title="Select Team Member">
    In project settings, find the member to update.
  </Step>

  <Step title="Change Permission Level">
    Click dropdown and select new role:
    - Promote: Viewer ‚Üí Commenter ‚Üí Editor ‚Üí Admin
    - Demote: Admin ‚Üí Editor ‚Üí Commenter ‚Üí Viewer

    Takes effect immediately.
  </Step>

  <Step title="Notify (Optional)">
    Send notification about permission change:
    - Explain reason for change
    - Clarify new capabilities
    - Answer questions

    Keeps team informed.
  </Step>
</Steps>

### Removing Team Members

<Warning>
  **Before Removing**:
  - Export their contributions if needed
  - Reassign any owned threads
  - Notify them if appropriate
  - Consider reducing permissions instead

  Removal is immediate and they lose all access.
</Warning>

<Steps>
  <Step title="Open Team Settings">
    Navigate to project settings ‚Üí Sharing tab
  </Step>

  <Step title="Select Member to Remove">
    Click menu (‚Ä¢‚Ä¢‚Ä¢) next to their name
  </Step>

  <Step title="Confirm Removal">
    Click **Remove from Project** and confirm

    <Info>
      Their past contributions remain in the project, but they can no longer access it.
    </Info>
  </Step>
</Steps>

## Notifications

Control how you're notified about team activity.

<AccordionGroup>
  <Accordion title="Notification Settings" icon="bell">
    Choose what to be notified about:
    - **@Mentions**: When someone tags you
    - **Comments**: On threads you're watching
    - **Changes**: To files you created
    - **Team Updates**: New members, permission changes

    Set frequency: Real-time, daily digest, or off
  </Accordion>

  <Accordion title="Thread Watching" icon="eye">
    Follow specific threads:
    - Auto-watch threads you create
    - Manually watch important threads
    - Unwatch when no longer relevant

    Get notified of all activity in watched threads.
  </Accordion>

  <Accordion title="Do Not Disturb" icon="moon">
    Pause notifications:
    - Set quiet hours
    - Snooze for set period
    - Mute specific projects

    Focus mode when you need it.
  </Accordion>
</AccordionGroup>
