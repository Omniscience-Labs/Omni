## What You Can Do

Once connected, agents can manage your Gmail account through natural conversation.

<CardGroup cols={2}>
  <Card title="Read Emails" icon="envelope-open">
    Check inbox, search messages, read threads
  </Card>
  <Card title="Send Messages" icon="paper-plane">
    Compose and send emails (see note about attachments below)
  </Card>
  <Card title="Organize" icon="folder">
    Apply labels, mark as read, archive, star
  </Card>
  <Card title="Filter & Search" icon="filter">
    Find emails by sender, subject, date, labels
  </Card>
</CardGroup>

<Warning>
  **Gmail Attachment Limitation**: The Gmail API currently does not support sending emails with file attachments through automation.
  
  **Workaround**:
  1. Agent can draft the email content and save it
  2. Go to the **Files panel** and download the attachment files
  3. Manually send the email through Gmail with downloaded attachments
  
  Alternatively, the agent can compose the email and you can copy the text to send manually with attachments.
</Warning>

## Setup

<Steps>
  <Step title="Connect Gmail">
    Navigate to **Integrations** and click **Connect Gmail**.

    {/* 
      üì∏ MEDIA PLACEHOLDER: Gmail Connect Button
      - Gmail card with Connect button highlighted
      - Shows icon, description, capabilities
      - Size: 1000x600px
    */}
  </Step>

  <Step title="Google OAuth">
    Log in and authorize:
    - Choose Google account
    - Review requested permissions
    - Click **Allow**

    <Info>
      Omni requests these permissions:
      - Read emails
      - Send emails
      - Manage labels
      - Modify messages (mark read, archive)
    </Info>
  </Step>

  <Step title="Configure Settings">
    **Optional Configuration**:
    - Enable/disable attachment access
    - Set default label handling
    - Configure notification preferences
    - Set up filters

    Defaults work for most users.
  </Step>

  <Step title="Assign to Agents">
    The Gmail connection (profile) is now available:
    - Shows in your list of integration profiles
    - Can be used by any agent
    - Create additional Gmail profiles for different accounts
    - Example: "Marketing Gmail", "Support Gmail", "Personal Gmail"

    <Check>
      Gmail connected! Test with: "What are my unread emails?"
    </Check>
  </Step>
</Steps>

<Info>
  **Multiple Gmail Accounts**: You can create multiple Gmail integration profiles. Name each profile clearly (e.g., "Marketing Email", "Support Email") and select which profile an agent should use in its integration settings.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't Send Emails" icon="ban">
    **Check**:
    - Send permission granted in OAuth
    - Gmail account has sending enabled
    - Not exceeding daily send limit (500/day)
    - Email format is correct

    Re-authenticate if needed.
  </Accordion>

  <Accordion title="Emails Not Found" icon="question">
    **Possible Issues**:
    - Search query too specific
    - Emails in different folder
    - Date filter too restrictive
    - Agent searching wrong account

    Try broader search terms.
  </Accordion>

  <Accordion title="Authentication Expired" icon="clock">
    **When Tokens Expire**:
    - Every 30-90 days (Google policy)
    - After password change
    - After permission modifications

    **Fix**: Click **Reconnect** in integrations page

    <Tip>
      Enable notifications for expiring connections.
    </Tip>
  </Accordion>
</AccordionGroup>
