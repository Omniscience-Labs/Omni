## How Integrations Work

<Steps>
  <Step title="Connect Service">
    Authenticate once using secure OAuth:
    - Click integration in Omni
    - Log in to the service
    - Authorize permissions
    - Connection established

    {/* 
      🎥 VIDEO PLACEHOLDER: Integration Flow (25 seconds)
      - Click "Connect Gmail" button
      - OAuth popup appears
      - User logs in (blur credentials)
      - Permission request shown
      - Success and redirect back
    */}

    <Info>
      Your credentials are encrypted and never stored in plain text.
    </Info>
  </Step>

  <Step title="Enable for Agents">
    Choose which agents can use the integration:
    - All agents (default)
    - Specific agents only
    - Custom per-agent permissions

    Different agents can have different access levels.
  </Step>

  <Step title="Agents Gain Tools">
    Integrated services provide new capabilities:
    
    **Gmail Connected** → Agents can:
    - Read emails
    - Send emails
    - Manage labels
    - Search inbox
    - Filter by criteria

    {/* 
      📸 MEDIA PLACEHOLDER: Agent Tools Post-Integration
      - Agent configuration showing new tools
      - Gmail tools section expanded
      - Each tool with description
      - Toggle switches for granular control
      - Size: 1600x900px
    */}
  </Step>

  <Step title="Use in Conversations">
    Simply ask agents to use connected services:
    ```
    "Check my unread Gmail messages"
    "Send a Slack message to #engineering"
    "Create a GitHub issue for this bug"
    ```

    Agents handle authentication and API calls automatically.

    <Check>
      No API keys, no coding - just natural conversation!
    </Check>
  </Step>
</Steps>

## Security & Permissions

<AccordionGroup>
  <Accordion title="OAuth Authentication" icon="shield-check">
    **Secure Connection Process**:
    - Industry-standard OAuth 2.0
    - No password sharing
    - Granular permission scopes
    - Revocable access tokens

    You control what Omni can do with each service.

    {/* 
      📸 MEDIA PLACEHOLDER: OAuth Permission Screen
      - Example Google OAuth permission request
      - Checkboxes showing requested permissions
      - "Allow" and "Deny" buttons
      - Security information
      - Size: 1200x900px
    */}
  </Accordion>

  <Accordion title="Permission Scopes" icon="key">
    **Control Access Level**:

    Example for Gmail:
    - ✓ Read emails (view only)
    - ✓ Send emails (compose and send)
    - ✗ Delete emails (not granted)
    - ✗ Modify labels (not granted)

    Grant only what your agents need.
  </Accordion>

  <Accordion title="Credential Management" icon="lock">
    **How Credentials Are Stored**:
    - Encrypted in database
    - Separate per user
    - Automatic token refresh
    - Revocable instantly

    **You Can**:
    - View connection status
    - Revoke access anytime
    - Re-authenticate if expired
    - Audit usage logs
  </Accordion>
</AccordionGroup>

<Warning>
  **Team vs Personal Integrations**: Some integrations are shared across your organization (e.g., company Slack), while others are personal (e.g., your Gmail). Check integration settings for details.
</Warning>

## Integration Health

Monitor and maintain your connections.

<Steps>
  <Step title="Check Status">
    View integration dashboard:
    - ✅ **Active**: Connected and working
    - ⚠️ **Warning**: Possible issues
    - ❌ **Error**: Connection failed
    - ⏸️ **Disabled**: Manually paused

    {/* 
      📸 MEDIA PLACEHOLDER: Integration Dashboard
      - List of all integrations with status
      - Health indicators (green/yellow/red)
      - Last successful sync time
      - Quick action buttons
      - Size: 1800x1000px
    */}
  </Step>

  <Step title="Review Usage">
    Track API calls and activity:
    - Requests per day
    - Most used endpoints
    - Error rates
    - Response times

    Identify issues early.
  </Step>

  <Step title="Resolve Issues">
    Common fixes:
    - Re-authenticate expired connections
    - Update permissions
    - Check service status
    - Review error logs

    Most issues resolve with re-authentication.
  </Step>
</Steps>
