## Setup

<Steps>
  <Step title="Connect Dropbox">
    Navigate to **Integrations** and click **Connect Dropbox**.

    {/* 
      ðŸ“¸ MEDIA PLACEHOLDER: Dropbox Connection
      - Dropbox integration card
      - Connect button and capabilities
      - OAuth flow preview
      - Size: 1200x800px
    */}
  </Step>

  <Step title="Dropbox Authorization">
    Grant access:
    - Log in to Dropbox account
    - Review folder access requested
    - Click **Allow**

    <Info>
      Omni can access files and folders you choose. Permissions are granular and revocable.
    </Info>
  </Step>

  <Step title="Select Folders">
    Choose which folders agents can access:
    - Full Dropbox access (all folders)
    - Specific folders only (recommended)
    - Team folders if applicable

    {/* 
      ðŸ“¸ MEDIA PLACEHOLDER: Folder Selection
      - Dropbox folder tree
      - Checkboxes for selection
      - Permission level per folder
      - Size: 1400x900px
    */}

    <Tip>
      Start with specific folders, expand access as needed.
    </Tip>
  </Step>

  <Step title="Test Integration">
    Verify connection:
    ```
    "List files in my Dropbox /Projects folder"
    "Upload this file to Dropbox"
    ```

    <Check>
      Dropbox connected! File automation ready.
    </Check>
  </Step>
</Steps>

## Automated Workflows

<Tabs>
  <Tab title="Backup Automation">
    **Regular Backups**:
    ```
    Schedule: Daily at midnight
    
    Task: "Backup important files:
    - Source: Agent workspace /critical/
    - Destination: Dropbox /Backups/{{date}}/
    - Compress as ZIP
    - Keep 30 days of backups
    - Delete older backups
    - Notify if backup fails"
    ```
  </Tab>

  <Tab title="Report Distribution">
    **Automated Delivery**:
    ```
    Trigger: Report generation complete
    
    Workflow:
    1. Upload report to Dropbox /Reports/
    2. Create share link (view-only)
    3. Send link via email to stakeholders
    4. Post announcement in Slack
    5. Log distribution activity
    ```

    Seamless report sharing.
  </Tab>

  <Tab title="File Monitoring">
    **Watch for Changes**:
    ```
    Monitor: Dropbox /Incoming/ folder
    
    When new file added:
    - Download to agent workspace
    - Process based on file type
    - Move to appropriate folder
    - Notify relevant team
    - Log processing"
    ```

    Reactive file processing.
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Upload Failed" icon="triangle-exclamation">
    **Common Causes**:
    - File size exceeds limits
    - Storage quota reached
    - Network interruption
    - Invalid filename characters

    **Solutions**:
    - Check Dropbox storage space
    - Split large files
    - Retry upload
    - Rename with valid characters
  </Accordion>

  <Accordion title="Permission Denied" icon="ban">
    **Access Issues**:
    - Folder not shared with integration
    - Read-only when write needed
    - Team folder restrictions

    **Fix**: Update folder permissions in Dropbox settings
  </Accordion>
</AccordionGroup>
