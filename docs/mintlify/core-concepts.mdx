## Overview

Omni is built around a few key concepts that work together to create powerful, autonomous AI agents. Understanding these fundamentals will help you get the most out of the platform.

## The Sandbox Environment

Every agent operates in a secure, isolated **sandbox** — a containerized environment where code executes and files are managed.

<Info>
  <strong>Chat Workspaces:</strong> Each chat you create has its own isolated
  workspace — think of it as giving the agent its own computer for that
  conversation. Files created in one chat are separate from other chats. This
  keeps work organized and prevents conflicts.
</Info>

### What's Inside the Sandbox?

<AccordionGroup>
  <Accordion title="File System" icon="folder-tree">
    A complete file system where agents can create, edit, and organize files. All files for a chat live in that chat's workspace.

    Access files through the <strong>Files panel</strong> on the right side of the chat interface. Download files to your computer or share them with others.

  </Accordion>

{" "}

<Accordion title="Code Execution" icon="code">
  Agents can run Python, Node.js, and shell commands safely within the sandbox.
  Perfect for data processing and automation.
</Accordion>

{" "}

<Accordion title="Browser Access" icon="browser">
  A headless Chromium browser for web automation, scraping, and testing. Agents
  can navigate websites just like you do.
</Accordion>

  <Accordion title="Internet Access" icon="globe">
    Agents can search the web, scrape content, and call APIs to gather real-time information.
  </Accordion>
</AccordionGroup>

### Security & Isolation

<Card title="Enterprise-Grade Security" icon="shield-check">
  Each agent runs in its own isolated container with no access to other users'
  data or environments. Automatic cleanup after sessions and secure credential
  management for integrations ensure enterprise-grade security.
</Card>

## Knowledge Base

The <strong>Knowledge Base</strong> is your agent's long-term memory — a collection of documents, files, and data that agents can reference.

### How It Works

<Steps>
  <Step title="Upload Documents">
    Drag and drop files or folders. Supported formats include PDF, Word, Excel,
    CSV, text files, code, and more.
  </Step>
  <Step title="Organize with Folders">
    Create a folder structure that makes sense for your team — by project,
    department, or topic.
  </Step>
  <Step title="Semantic Search">
    Agents use AI-powered search to find relevant information based on meaning,
    not just keywords.
  </Step>
  <Step title="Automatic Context">
    When agents need information, they automatically search the knowledge base
    and pull in relevant context.
  </Step>
</Steps>

<Tip>
  Upload your company's documentation, reports, and resources to give agents
  deep context about your business.
</Tip>

## Agents

<strong>Agents</strong> are autonomous AI workers configured with specific capabilities,
knowledge, and integrations.

<CardGroup cols={2}>
  <Card title="Custom Configuration" icon="sliders">
    Each agent has its own set of enabled tools, connected services, and
    knowledge base access
  </Card>
  <Card title="Specialized Roles" icon="user-tie">
    Agents can be specialized for research, content creation, data analysis, or
    custom workflows
  </Card>
  <Card title="Marketplace Ready" icon="store">
    Share your agents with the team or install pre-built agents from the
    marketplace
  </Card>
  <Card title="Automatic Task Breakdown" icon="list-check">
    Agents automatically break complex requests into smaller tasks and show
    their plan
  </Card>
</CardGroup>

<Info>
  When you give an agent a complex task, it automatically breaks it into smaller
  steps using its own reasoning — not pre-programmed rules.
</Info>

### Agent Types

<Tabs>
  <Tab title="General Purpose">
    Versatile agents that handle a wide variety of tasks.
    <strong>Best for:</strong> Ad-hoc requests, exploratory work, general assistance
  </Tab>
  <Tab title="Specialized">
    Agents configured for specific domains.
    <strong>Best for:</strong> Recurring workflows, domain expertise, consistent
    output
  </Tab>
  <Tab title="Integrated">
    Agents connected to external services.
    <strong>Best for:</strong> Automation, notifications, synchronization
  </Tab>
</Tabs>

## Projects & Threads

<strong>Projects</strong> organize your work.
<strong>Threads</strong> are individual conversations within a project.

### Projects

- Marketing Campaign
- Q4 Research
- Customer Onboarding

### Threads

Each thread lets you:

- Assign tasks and track progress
- View files and outputs
- Inspect tool usage
- Continue conversations over time

<Info>
  Create separate threads for different tasks to keep context clean and focused.
</Info>
